# Установка исправленной версии в VS Code

> Для апгрейда V2 сначала прочитайте `UPGRADE_V2_RU.md`. Если защищённая версия уже установлена и SQL выполнялся, повторная настройка RLS не требуется.

## 1. Подготовка

1. Сделайте резервную копию текущей папки проекта.
2. Распакуйте архив в новую папку и откройте её через **File → Open Folder** в VS Code.
3. Скопируйте `.env.example` в `.env.local` и вставьте URL и anon key из Supabase: **Project Settings → API**.

## 2. Установка и проверка

Откройте **Terminal → New Terminal** и выполните:

```bash
npm ci
npm run lint
npm run build
npm run dev
```

Откройте `http://localhost:3000/ru`. Проверьте также `/uz`, `/en`, поиск и мобильное меню.

## 3. Обязательная защита Supabase

1. Откройте Supabase → **SQL Editor**.
2. Откройте файл `supabase/security.sql` в VS Code.
3. Замените `administrator@example.com` на email настоящего администратора.
4. Выполните SQL целиком. Затем выйдите из админ-панели и войдите снова, чтобы JWT получил роль.
5. Проверьте в приватном окне: `/ru/admin` должен перенаправлять на страницу входа.

Важно: не помещайте service-role key в `.env.local` или Netlify. Сайт использует только публичный anon key, а изменение данных защищает RLS.

## 4. Публикация на Netlify

В **Site configuration → Environment variables** добавьте три переменные из `.env.example`. Затем загрузите проект в GitHub и подключите репозиторий к Netlify либо запустите **Deploys → Trigger deploy → Clear cache and deploy site**.

Build command: `npm run build`. Для Next.js Netlify определит publish-настройки автоматически.
