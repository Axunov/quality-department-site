# Апгрейд сайта V2

## Что добавлено

- автоматическая светлая и тёмная тема с сохранением выбора;
- переключатель темы в публичной шапке и админ-панели;
- тема применяется до загрузки React, поэтому страница не мигает;
- улучшенная полупрозрачная шапка и мобильные элементы управления;
- ссылка быстрого перехода к содержимому для клавиатурной навигации;
- тёмные стили для страниц, карточек, текста, редакционных материалов и админки;
- новый адаптивный административный дашборд;
- цветовое разделение новостей, документов и сотрудников;
- индикатор рабочего состояния и обновлённые быстрые действия;
- закрепляемая боковая панель администратора на больших экранах.

## Переход с исправленной версии

1. Сделайте резервную копию текущего проекта.
2. Распакуйте V2 в отдельную папку.
3. Перенесите свой `.env.local` из предыдущего проекта или создайте его по `.env.example`.
4. Не переносите старые `node_modules` и `.next`.
5. Выполните:

```bash
npm ci
npm run lint
npm run build
npm run dev
```

6. Проверьте `/ru`, `/uz`, `/en`, мобильное меню, переключатель темы и `/ru/admin`.
7. SQL повторно выполнять не требуется, если `supabase/security.sql` уже выполнялся для предыдущей исправленной версии.
8. Отправьте изменения в GitHub и в Netlify выберите **Clear cache and deploy site**.

## Важное ограничение

Статусы обращений, журнал действий и реальные графики мониторинга требуют согласованной новой схемы базы данных. В этой версии интерфейс и безопасность улучшены без рискованного изменения существующих таблиц и данных. Их следует внедрять отдельной миграцией после утверждения полей, ролей и рабочего процесса отдела.
