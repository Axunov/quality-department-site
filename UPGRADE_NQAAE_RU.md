# Полный апгрейд по результатам сравнения с nqaae.uz

## Что было взято за ориентир

Сайт Национального агентства хорошо разделяет аккредитацию, открытые данные, исследования, FAQ и обращения. В проект отдела перенесена эта логика, но без перегруженного многоуровневого меню, повторяющихся региональных блоков и неподтверждённых цифр.

## Изменения проекта

- на главной добавлен центр быстрых сервисов из шести направлений;
- навигация получила отдельное раскрывающееся меню «Система качества»;
- добавлена полноценная страница самооценки с шестью этапами;
- добавлен раздел открытых материалов и принцип доказательности показателей;
- меню адаптировано для компьютера и телефона на русском, узбекском и английском языках;
- новые разделы добавлены в sitemap для индексации;
- исправлено поведение формы входа: пользователь без роли администратора теперь сразу видит понятную ошибку, а не незаметный возврат на страницу входа;
- сохранены тёмная тема, режим доступности, поиск, Supabase и действующая защита админки.

## Установка

1. Сделайте резервную копию текущего проекта.
2. Распакуйте новый архив в отдельную папку.
3. Скопируйте свой `.env.local` из действующего проекта.
4. Не переносите папки `node_modules` и `.next`.
5. Выполните `npm ci`, `npm run lint`, `npm run build`, затем `npm run dev`.
6. Проверьте `/ru`, `/ru/self-assessment`, `/ru/open-data` и вход в `/ru/admin/login`.
7. Отправьте проект в GitHub и выполните в Netlify: **Clear cache and deploy site**.

## Проверка администратора

В `auth.users.raw_app_meta_data` у администратора должно быть `{ "role": "admin" }`. После назначения роли необходимо выйти, очистить cookies сайта и войти снова.
